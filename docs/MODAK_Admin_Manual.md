## 1. 시작하기 — 로그인·로그아웃

### 접속

브라우저에서 `modak_admin.html` 파일을 열면 로그인 화면이 표시됩니다.  
Firebase Hosting 배포 후에는 지정된 URL로 접속합니다.

### 데모 계정 (Firebase 연동 전)

| 이메일 | 비밀번호 | 권한 |
|--------|---------|------|
| `admin@modak.com` | `tkfkdgo7!` | 슈퍼 관리자 |
| `juystyle@modak.com` | `tkfkdgo7!` | 슈퍼 관리자 |
| `juystyle1@modak.com` | `tkfkdgo7!` | 관리자 |
| `guest@modak.com` | `tkfkdgo7!` | 읽기전용 |

### 로그인

1. 이메일과 비밀번호 입력 (`Enter` 키 지원)
2. **로그인** 버튼 클릭
3. 성공 시 대시보드로 이동, 좌측 하단에 프로필 표시

### 로그아웃

- 좌측 하단 **로그아웃** 버튼 클릭
- 또는 헤더 우상단 ⚙️ 설정 패널 → **로그아웃**

> ⚠️ Firebase 연동 후에는 데모 계정을 삭제하고 실제 계정으로 교체하세요.

## 2. 권한 체계

| 권한 | 이름 | 설명 |
|------|------|------|
| `super` | 슈퍼 관리자 | 모든 기능 사용 가능. 관리자 계정 관리 포함. |
| `manager` | 관리자 | 콘텐츠·회원·광고 관리. 관리자 계정 관리 불가. |
| `viewer` | 읽기전용 | 조회만 가능. 모든 수정·처리 불가. |

### 읽기전용(viewer) 제한

읽기전용 계정으로 처리 버튼 클릭 시 → **"🔒 읽기전용 계정은 이 작업을 할 수 없습니다"** 토스트 표시. 데이터 변경 없음.

제한 항목: 모든 신고·게시물·회원·스폰서·딜·주문·공지 처리 / 관리자 초대·변경·삭제 / 푸시 알림 발송 / 지역·기능 플래그 토글

## 3. 대시보드

로그인 후 첫 화면. 서비스 전체 현황을 한눈에 파악합니다.

### KPI 카드 (4개)

| 카드 | 내용 |
|------|------|
| 누적 회원 | 전체 가입 회원 수 + 오늘 신규 가입 수 |
| 오늘 방문자 | 당일 DAU + 전일 대비 증감률 |
| 이번달 광고수익 | 월 누적 광고 수익(USD) + 전월 대비 |
| 미처리 신고 | 처리 안 된 신고 건수 (빨간색 강조) |

### 주요 위젯

- **신규 가입 차트**: 30일 일별 막대 차트. 마우스 오버 시 수치 표시. 오늘 바는 파란색 강조.
- **미처리 신고 목록**: 최근 5건 요약. **전체 보기** 클릭 시 신고 관리로 이동.
- **실시간 활동 피드**: 신규 가입·게시물 등록·신고 접수 등 최근 활동.
- **스폰서 슬롯**: Premium 고정 슬롯 4개 현황 (업체명, 만료 D-days).
- **게시물 현황**: 카테고리별 게시물 수.

## 4. 신고 관리

🚨 **가장 먼저 매일 확인해야 할 메뉴입니다.**

### 탭

| 탭 | 내용 |
|----|------|
| 미처리 | 처리 전 신고 (빨간 숫자 배지) |
| 처리완료 | 처리된 신고 이력 |
| 전체 | 전체 신고 목록 |

### 목록 정보

심각도 점(🔴 높음 / 🟡 중간 / ⚫ 낮음) + 제목 + 유형 배지 + 신고 시간

### 처리 방법

신고 항목 클릭 → 우측 상세 패널 확인 → 처리 버튼 선택:

| 버튼 | 동작 |
|------|------|
| 통과 허용 | 신고를 무혐의로 종결 |
| 게시물 삭제 | 해당 게시물 삭제 |
| 작성자 경고 | 사용자에게 경고 조치 |
| 작성자 정지 | 해당 계정 즉시 정지 |

처리 완료 후 자동으로 처리완료 탭으로 이동하며 사이드바 배지 숫자가 감소합니다.

## 7. 스폰서 관리

### Premium 고정 슬롯 (4개)

홈 화면 상단에 항상 노출되는 프리미엄 광고 슬롯.  
만료 D-7 이하: 빨간 배지 / D-14 이하: 주황 배지.

### 처리 방법

업체 선택 → 우측 상세 패널:

| 버튼 | 동작 |
|------|------|
| 기간 연장 | 일수 입력 → 종료일 연장 |
| 검증 완료 | ✓ 검증 배지 부여 (토글) |
| Premium / NEW 태그 | 태그 설정 |
| 업체 삭제 | 영구 삭제 |

### 신규 업체 등록

**+ 업체 등록** → 폼 작성:
1. 업체명 / 이모지 / 플랜 선택 (Premium $350/월 또는 일반 $150/월)
2. 태그 / 광고 기간 / 연락처 / 주소 / 업체 소개
3. **업체 등록하기**

## 8. 딜·혜택 관리

### KPI 카드

전체 딜 수 / 진행 중 / 만료 임박 / 총 쿠폰 사용 수

### 목록

**필터**: 전체 / 진행중 / 만료임박 / 만료됨  
사용량 프로그레스 바: 90% 이상 빨강 / 70% 이상 주황

### 처리 방법

| 버튼 | 동작 |
|------|------|
| 기간 연장 | 일수 입력 → 종료일 연장 |
| 한도 변경 | 사용 한도 수량 변경 |
| 일시 중단 | 딜 비활성화 |
| 재개 | 다시 활성화 |
| 딜 삭제 | 영구 삭제 |

### 신규 딜 등록

**+ 딜 등록** → 폼 작성:
1. 딜 제목 / 이모지 / 카테고리
2. 할인 유형: % 할인 / $ 할인 / 무료 제공 / 증정 이벤트
3. 쿠폰 코드 / 사용 한도 / 연결 스폰서
4. 기간 / 지역 / 최소 주문 조건 / 설명

## 13. 푸시 알림

### 작성 탭

1. 대상 지역 칩 선택 (전체 / 특정 지역)
2. 알림 제목 (최대 50자) / 내용 (최대 100자)
3. 발송 시간: **즉시** 또는 **예약** (날짜·시간 선택)
4. **테스트 발송** → 내 기기 테스트
5. **전송하기** → 발송

실시간 미리보기: 입력 즉시 핸드폰 화면에 반영.

### 발송 이력 탭

발송 건별 제목 / 대상 / 발송 수 / **오픈율** (프로그레스 바) / 발송 일시.

## 16. 지역·서비스 설정

### KPI 카드

운영 중인 지역 / 오픈 예정 / 전체 지역 회원 수 / 활성 기능 플래그

### 서비스 지역

각 지역 카드의 **ON/OFF 토글**로 서비스 활성화·비활성화.

> ⚠️ OFF 처리 시 해당 지역 사용자 서비스 이용 불가

**새 지역 추가**: `+` 카드 클릭 → 지역명·주(State)·오픈 예정일 입력.
